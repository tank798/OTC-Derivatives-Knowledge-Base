# 抓取与知识库规则

检索日期：2026-07-07

## 1. 全量边界

本项目的“全量”定义为：

1. 覆盖中国金融监管官方源和市场基础设施官方规则源。
2. 对每个官方源保留可重复执行的抓取入口、分页方式、过滤规则和缺口状态。
3. 对已抓取文件保留原始文件、抽取文本、文档级元数据和条款级切片。
4. 对监管文件新增、废止、修订进行增量更新，而不是维护一次性静态清单。

不把非官方转载、媒体解读、公众号汇总作为正式法规依据。它们可以进入线索池，但不能直接支持“能不能做”的结论。

## 2. 官方源优先级

### S0 上位法与行政法规

- 国家法律法规数据库。
- 全国人大、国务院正式发布渠道。

### S1 金融监管部门

- 中国证监会及证券期货法规数据库。
- 中国人民银行。
- 国家金融监督管理总局及原银保监会公开规则。当前使用 `/cbircweb/DocInfo/SelectItemAndDocByItemPId` 和 `/cbircweb/DocInfo/SelectByDocId` 官方接口读取列表和正文。
- 国家外汇管理局。

### S2 自律组织与市场基础设施

- 中国证券业协会。
- 中国证券投资基金业协会。
- 中国银行间市场交易商协会。
- 证券交易所、期货交易所。
- 中证登、中债登、上海清算所、外汇交易中心/中国货币网。中国货币网当前使用 `/ags/ms/cm-s-notice-query/contents` 和 `/ags/ms/cm-s-notice-query/txtAttachmentInfo` 官方接口。

### S3 业务解释与问答

- 官方业务指南、问答、通知、纪律处分、备案规则、名单公示。
- 可用于辅助解释，但优先级低于法律、规章和规范性文件。

## 3. 文档级字段

每个法规/规则文件至少保留：

- `doc_id`: 稳定 ID，建议由来源 ID 和 URL 哈希生成。
- `source_id`: 官方源 ID。
- `publisher`: 发布主体。
- `title`: 文件标题。
- `url`: 原始 URL。
- `retrieved_at`: 抓取时间。
- `published_at`: 发布日期。
- `effective_at`: 生效日期。
- `status`: `effective` / `repealed` / `amended` / `unknown`。
- `authority_level`: `law` / `administrative_regulation` / `department_rule` / `normative_doc` / `self_regulatory_rule` / `exchange_rule` / `business_guideline` / `unknown`。
- `asset_classes`: 股票、固收、基金、外汇、商品、信用、跨资产等。
- `product_types`: 场外期权、收益凭证、互换、远期、回购、ABS、REITs、资管计划等。
- `regulated_entities`: 证券公司、基金管理人、私募管理人、期货公司、银行、保险、信托、投资者等。
- `body_read`: 是否读取正文。
- `raw_path`: 原始文件路径。
- `text_path`: 抽取文本路径。
- `checksum`: 原始文件哈希。

## 4. 条款级字段

条款切片保留：

- `clause_id`: 文档 ID + 条款序号。
- `doc_id`: 所属文件。
- `article_no`: 条号，不能识别时为空。
- `heading_path`: 章/节/条标题路径。
- `text`: 条款正文。
- `citations`: 原始 URL、标题、发布主体、日期。
- `compliance_tags`: 适当性、准入、禁止性规定、信息披露、风险控制、备案、报告、交易限制、清算结算、跨境限制等。
- `retrieval_keywords`: 用于检索的同义词和业务词。

## 5. 条款切分规则

1. 优先按中文条款标记切分：`第一条`、`第二条`、`第十条`、`第1条`。
2. 保留章、节、附件标题，作为 `heading_path`。
3. 如果文件不是条款式文本，按段落分块，每块约 800 到 1500 中文字符，保留 100 到 200 字重叠。
4. 不截断表格、名单、备案字段等结构化内容；无法安全切分时整段保留。
5. 每个条款块必须能追溯到文档级元数据和原始 URL。

## 6. RAG 回答规范

回答“产品能不能做”时，模型必须按以下顺序检索和引用：

1. 上位法和行政法规。
2. 证监会、央行、金监总局、外汇局部门规章/规范性文件。
3. 自律规则和交易场所业务规则。
4. 备案、名单、公示、问答、处罚和监管案例。

输出必须包含：

- 结论：可做 / 不可做 / 有条件可做 / 需要人工合规复核。
- 命中的规则条款。
- 限制条件和需补充确认的问题。
- 引用：文件名、条号、发布主体、原始链接、发布日期/生效日期。

禁止只给泛泛判断，不给条款依据。

## 7. 缺口处理

以下情况写入 `data/processed/gaps.jsonl`：

- 官方页面 403、TLS、动态渲染或验证码导致无法抓取。
- 找到目录但未能读取正文。
- 只有搜索摘要，没有正文。
- 文件疑似被废止或修订，但未能确认效力状态。
- 同名文件存在多个版本，无法确认适用版本。
