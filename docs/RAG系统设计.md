# RAG 系统设计

目标：回答“某个场外衍生品/跨资产产品能不能做、需要满足什么条件、依据哪条法规”。

## 1. 数据层

### 原文层

- 保存路径：`data/raw/html/`、`data/raw/pdf/`、`data/raw/text/`
- 作用：保留官方原文，方便审计和重新切分。

### 文档索引层

- 保存路径：`data/processed/documents.jsonl`
- 作用：记录所有抓取到的页面和文件。
- 注意：该层可能包含目录页、通知页和少量新闻页，不能直接作为高置信法规依据。

### 监管文档层

- 保存路径：`data/processed/regulatory_documents.jsonl`
- 作用：从文档索引中筛出更像法规/规则/通知/指引的文件。

### 证据账本层

- 保存路径：`data/processed/evidence_ledger.jsonl`
- 作用：存放人工/Agent 核验过的关键规则，是高置信入口。
- 字段区分 `body_read` 和 `attachment_read`，防止把只读公告误当作附件正文已核验。

### 条款层

- 保存路径：`data/processed/clauses.jsonl`
- 作用：面向检索增强生成的条款级切片。

## 2. 检索策略

产品合规判断按两阶段检索：

1. 高精度：先检索 `evidence_ledger.jsonl`，确定核心规则和监管层级。
2. 高召回：再检索 `clauses.jsonl` 和 `regulatory_documents.jsonl`，补充条文和细项。

优先级：

1. 法律、行政法规、部门规章。
2. 监管规范性文件。
3. 自律规则。
4. 交易所、登记结算、清算机构业务规则。
5. 公告、问答、模板、名单、备案公示。

```text

   原文链接：……

限制条件：
- ……

- ……
```

## 4. 风险控制

- 未命中证据账本时，模型不得直接给确定结论。
- 命中“公告已核验_附件未核验”的文件时，必须提示附件未核验。
- 对跨境、外汇、资管、私募、银行间、场外期权、收益互换、信用衍生品等问题，至少同时检索监管部门和自律/基础设施规则。
- 对 2026-11-16 前后的场外衍生品问题，需要特别处理《衍生品交易监督管理办法（试行）》的过渡安排。
